# Support

Thanks for your interest in the **Ultimate Drupal Reference**.

This repository is a curated reference for modern Drupal developers and site builders.
It is **not** a general Drupal support desk, and it is not intended to provide one-to-one troubleshooting for individual projects.

## What this repository can help with

Please use this repository's **Issues** or **Discussions** for things like:

- broken links
- outdated recommendations
- missing modern Drupal resources
- suggestions for improving the README structure or scope
- questions about whether a resource fits this repository

## Where to post what

### Use Issues for

Open an **Issue** if you found something concrete and actionable, such as:

- a broken link
- outdated content
- a section that needs updating
- a specific improvement to the repository

### Use Discussions for

Use **Discussions** for:

- broader ideas
- questions about scope or curation
- suggestions that are not yet ready to become an issue or pull request
- general feedback about the repository

### Use Pull Requests for

Open a **Pull Request** if you already know the change is straightforward and are ready to propose it directly.

## What this repository does not provide

Please do **not** use this repository for:

- general Drupal support questions
- debugging your specific Drupal project
- module support requests unrelated to this repository
- server, hosting, or DevOps support for your own setup
- private consulting requests

## Where to get general Drupal help instead

If you need general Drupal help, these are better places to ask:

- [Drupal Slack](https://www.drupal.org/join-slack)
- [Drupal Answers](https://drupal.stackexchange.com/)
- [Drupal forums](https://www.drupal.org/forum)
- [Drupal documentation](https://www.drupal.org/documentation)

## Security

If your concern is about a Drupal core or contributed project security issue, please follow the official Drupal security process rather than posting it here:

- [Drupal Security Team](https://www.drupal.org/security)

If the issue is specific to this repository itself, open an issue unless there is a strong reason not to.

## Code of Conduct

Please note that participation in this repository is governed by the [Code of Conduct](CODE_OF_CONDUCT.md).

## Final note

The best support requests for this repository are the ones that help keep it:

- current
- practical
- high-signal
- useful for modern Drupal work

Thanks for helping improve it.
